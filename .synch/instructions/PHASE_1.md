# PHASE 1 指令卡：架构奠基

> **目标**: 完成项目骨架，冻结契约  
> **输入**: 用户需求描述  
> **输出**: 可进入 Phase 3 开发的完整契约与目录结构  
> **禁止**: 任何实现代码

---

## 执行路径

```
D:\vcp\projects\Synch\Synch项目流程.md → 参考 Phase 1 章节
```

---

## 步骤清单 (必须顺序执行)

### Step 1: 需求澄清 (5分钟)
- 追问用户：边界场景、技术约束、质量属性优先级
- 输出：`docs/requirements/确认书.md` (含用户签字栏)

### Step 2: 目录结构 (10分钟)
- 按规范创建完整目录树
- 每个模块目录下放置 `.gitkeep` 和 `DEPENDS_ON` 文件
- 输出：物理目录结构

### Step 3: README (10分钟)
- 强制章节：快速开始、架构概览、模块索引、测试策略
- 输出：`README.md`

### Step 4: 契约定义 (核心，20分钟)
- 为每个模块创建 `contracts/modules/[name].contract.ts`
- 必须包含：领域模型、输入/输出类型、服务接口、事件定义、依赖声明
- 标记 `@frozen true` 和版本号
- 输出：契约文件 + `contracts/index.ts` 统一导出

### Step 5: CI 配置确认 (5分钟)
- 确认 `.github/workflows/synch-ci.yml` 存在
- 无需修改，仅确认

---

## 自检清单 (完成前必须全部勾选)

- [ ] `确认书.md` 有用户签字确认
- [ ] 所有契约文件标记 `@frozen true`
- [ ] 契约副本存入 `.synch/contracts/v1.0/`
- [ ] 目录结构符合规范 (含 `.github/`、`.synch/`、`contracts/`、`src/modules/`)
- [ ] README 包含模块索引和测试策略章节
- [ ] 无任何实现代码 (仅 TypeScript 类型定义)

## 紧急中止条件

如遇以下情况，立即停止并上报：
- 用户需求存在根本性矛盾
- 技术栈与现有 CI 配置不兼容
- 发现安全风险 (密钥、注入等)
