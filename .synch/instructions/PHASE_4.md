# PHASE 4 指令卡：集成验证

> **目标**: 验证所有模块集成，生成发布候选  
> **输入**: 各模块 API 摘要 + CI 产物  
> **输出**: 集成报告 + 发布候选包  
> **禁止**: 加载任何模块的 internal/ 实现

---

```

```

---

## 步骤清单

### Step 1: 收集 CI 状态
- 从 GitHub API 获取所有模块最新 CI 状态
- 确认：contract-compliance ✅、module-unit-tests ✅、integration-tests ✅

### Step 2: 依赖图验证
- 输入：各模块 API 摘要中的依赖声明
- 输出：模块依赖有向图 (DAG 检测)
- 失败：发现循环依赖 → 生成重构任务

### Step 3: 契约兼容性验证
- 对比各模块契约的输入/输出类型
- 输出：类型兼容性报告

### Step 4: 事件流验证
- 梳理各模块发布/订阅的事件
- 输出：事件总线拓扑图

### Step 5: 性能基线对比
- 从 CI 产物获取 `performance-results`
- 对比 p50/p95/p99 延迟
- 输出：性能回归检测报告

### Step 6: 生成集成报告
输出：`docs/reports/integration-[日期].md`

## 冲突解决授权

| 冲突类型 | 解决方式 | 需用户确认? |
|:---|:---|:---|
| 契约不匹配 | 触发变更流程 | 是 |
| 循环依赖 | 提取共享契约 | 否 |
| 性能回归 | 生成优化任务 | 是 (若 >20%) |
| 集成测试失败 | 定位模块 → 修复任务 | 否 |
