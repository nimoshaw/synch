# Synch 交互权限与契约架构 - 下阶段开发计划 (Step 5)

当前已完成核心安全原语 (Rust) 与中继路由重构 (Go)。Step 5 的重点是**打通全链路闭环**，实现客户端的自动化握手、数据持久化以及更高级的安全特性。

## 一、 核心任务清单

### 1. 客户端握手机制 (Handshake Closing)
- **目标**：实现 `BIND_REQ` / `BIND_ACK` 的自动化流程。
- **开发点**：
    - 在 `synch-sync` 中集成 `ContractManager`。
    - 实现挑战-响应逻辑：A 发起请求 -> B 自动验证/手动弹出确认 -> B 签署并返回 -> A 激活并同步至中继。

### 2. 中继服务器持久化 (Persistence)
- **目标**：解决中继重启后契约与离线消息丢失的问题。
- **开发点**：
    - 为 Go Relay 引入 BadgerDB 或 SQLite 存储后端。
    - 持久化 `Contract` 摘要信息。
    - 持久化 `OfflineMessage` 队列（加密态存储）。

### 3. 感知层级逻辑集成 (Perception & Discovery)
- **目标**：实现 L0-L4 的隐私过滤。
- **开发点**：
    - 根据用户设置的感知层级，动态控制 `PresenceUpdate` 的广播范围。
    - 实现 L4 (隐身模式)：中继仅允许来自“已有契约节点”的路由探测。

### 4. 完整双棘轮算法 (Full Double Ratchet)
- **目标**：从对称棘轮升级为完整的异步棘轮。
- **开发点**：
    - 集成 Diffie-Hellman 棘轮：定期在交互中插入新的 X25519 公钥，强制刷新根密钥。
    - 增强自愈能力：即使一条消息的链密钥泄露，下一次 DH 交换后安全性将自动恢复。
