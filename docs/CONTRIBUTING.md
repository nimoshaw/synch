# Contributing to Synch

Thank you for your interest in contributing to Synch! We welcome contributions from everyone.

## 🤝 Code of Conduct
Please be respectful and professional in all interactions.

## 🛠️ Development Workflow

1. **Fork & Clone**: Fork the repository and clone it locally.
2. **Setup**: Run `task init` to install dependencies.
3. **Branch**: Create a feature branch (`feat/your-feature` or `fix/your-fix`).
4. **Code**: Implement your changes following our coding standards.
5. **Test**: Run `task test:all` to ensure everything is working.
6. **PR**: Submit a Pull Request with a clear description of your changes.

## 📝 Coding Standards

- **Rust**: Follow `clippy` and standard `rustfmt` rules.
- **Go**: Use `gofmt` and standard Go conventions.
- **TypeScript**: Use Prettier and ESLint (as configured in the project).
- **Protobuf**: Keep definitions backward compatible.

## 💬 Communication
For major changes, please open an issue first to discuss the design.
