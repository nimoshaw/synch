# SYNCH 项目需求确认书 (Phase 1)

## 1. 项目概览
- **项目名称**: SYNCH 心契协议 v2.0
- **核心目标**: 实现去中心化 Agent 通讯与多端 Vault 数据同步。

## 2. 核心技术约束
- **加密方案**: Ed25519 (身份), X25519 (加密), AES-GCM (存储)。
- **跨平台**: Rust FFI (UniFFI) 提供跨各端核心逻辑。
- **协议**: 基于 WebSocket + Protobuf v3。

## 3. 质量属性优先级
1. **安全性**: 数据必须端到端加密，密钥由用户掌握。
2. **可用性**: 支持离线编辑与后期同步（CRDT/Delta）。
3. **扩展性**: 支持多服务端联邦与第三方插件。

## 4. 边界场景确认
- [ ] 弱网环境下的重连与断点续传。
- [ ] 多端并发修改冲突解决（LWW/手动）。
- [ ] 证书过期与吊销机制。
